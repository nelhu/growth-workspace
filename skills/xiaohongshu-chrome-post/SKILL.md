# 小红书内置浏览器发布

把一次发布拆成“可审阅内容包”和“浏览器填写”两个阶段。公开发布会影响用户账号，因此即使用户已经给了素材，也只在最终点击发布前请求一次当次确认。

## 输入与边界

- 接受用户提供的图片、标题/主题、正文要点、话题、首评和期望发布时间；缺省时生成草稿，但不要把虚构经历写成亲身体验。
- 优先使用用户自有图片。若用户要求“真实景点图”，只使用来源与许可可核验的实拍图，并保存来源页、原图链接、许可和作者信息；不可把 AI 图标记为实拍。
- 不抓取或转存受限素材；不伪造打卡、消费、住宿或人物亲历。
- 绝不尝试绕过登录、验证码、滑块、风控、频率限制或网站规则。遇到这些情况停下，请用户自行完成。
- 仅操作用户已登录的 Codex 内置浏览器；不要读取或暴露 cookie、密码、验证码或私密消息。

## 阶段 1：生成内容包

1. 先确认本次是草稿、预览，还是用户已明确要求发布。没有明确发布意图时，到内容包为止。
2. 生成以下字段：标题、正文、话题、首评、图片绝对路径和图片来源信息。正文长度、话题数等有用户约束时精确校验。
3. 图片与文案的地点、时段和体验必须一致。对真实景点，优先以官方旅游机构或景点官网校验名称与基础事实。
4. 将内容包写入当前工作区 `drafts/xiaohongshu-<slug>/post.json`，使用以下结构：

```json
{
  "status": "ready_for_review",
  "should_publish": false,
  "title": "",
  "body": "",
  "topics": ["#话题一"],
  "first_comment": "",
  "images": ["/absolute/path/image.jpg"],
  "image_sources": [{
    "source_page": "https://...",
    "original_url": "https://...",
    "license": "CC0",
    "author": ""
  }]
}
```

5. 展示发布预览：标题、正文、全部话题、首评和图片缩略/路径。明确说明“未发布”。

## 阶段 2：通过 Codex 内置浏览器填写

只有用户明确要求发布且确认内容包后才进入本阶段。操作前先加载并遵循 `browser:control-in-app-browser` Skill，使用 `iab` 浏览器绑定；不要改用用户本机 Chrome，除非用户明确要求切换。

1. 使用用户已打开的内置浏览器标签；若没有可接管标签，打开 `https://www.xiaohongshu.com/`，从官网可见入口进入创作页。不要把登录链接、账号或页面状态发送到其他系统。
2. 上传 `images` 中的文件，填写标题、正文和话题。若页面自动格式化内容，回读可见预览，确认标题、正文、话题和图片数量没有丢失。
3. 填写首评的前提是平台提供正常的评论入口；否则在发布后停止并说明需要用户手动补充，不能伪造评论已发布。
4. 到达最终“发布”按钮后，逐项回报即将公开的内容，并询问：`内容已填好，是否现在公开发布？`
5. 仅在用户对这一次确认回复肯定后点击“发布”。发布后回读成功提示或作品列表的可见状态，记录时间和公开链接（若页面可见）。

## 失败与恢复

- 上传失败、编辑器变化、平台验证或网络异常：保留内容包，说明当前页面和未完成步骤；不要反复点击发布。
- 内置浏览器若无法取得可用的 DOM 快照或截图、无法唯一定位控件，立即停止填写并保留草稿；不要猜测坐标、盲点“发布”或切换到本机 Chrome。
- 用户改变标题、正文、图片或首评后，重新预览；之前的“现在发布”确认作废，必须再次确认。

## 最终回复

草稿阶段回复：内容包路径、标题、正文、话题、首评、图片路径/来源、状态“未发布”。

发布阶段回复：上述信息加最终状态（已发布 / 未发布）、可见链接（如有）和任何需要用户手动完成的步骤。
